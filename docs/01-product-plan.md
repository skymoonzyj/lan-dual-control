# 产品策划书

## 1. 产品定位

项目暂定名：局域网双端远控。

这是一个面向个人和小型工作环境的局域网远程控制工具，重点解决以下问题：

- 用户想从 Windows 窗口化控制 Mac。
- 用户希望中文界面，不想长期依赖英文远控软件。
- 用户希望控制链路只走局域网，不经过外网中继或账号服务器。
- 用户希望控制端能听到被控端声音。
- 用户希望能调节远程画面的分辨率、刷新率、带宽、画质、窗口化或全屏显示。
- 用户希望剪贴板能同步文字，也能同步文件、压缩包等常见文件。
- 用户希望未来可以反过来从 Mac 控制 Windows，并支持一键反控。

产品不追求第一版就达到 Parsec 的商业级低延迟体验，而是先做一个稳定、可理解、可维护的个人可用工具。

## 2. 目标用户

- 有 Windows 主力机和 Mac mini 的个人用户。
- 希望局域网内跨系统操作，不想频繁切换键鼠和显示器的用户。
- 需要中文界面和明确连接状态提示的用户。
- 希望项目能被两个 Codex 环境分别在 Windows 和 Mac 上继续开发的用户。

## 3. 核心场景

### 场景 A：Windows 控制 Mac

用户在 Windows 上打开控制端，选择局域网内的 Mac，输入配对码或固定密码后，在一个普通窗口中看到 Mac 桌面并进行鼠标键盘操作。

### 场景 B：窗口化和全屏切换

用户可以让远控画面保持窗口化，也可以切换到全屏。窗口化时支持拖动、缩放、最小化；全屏时支持快捷键退出。

### 场景 C：画面和带宽调节

用户可以根据局域网环境选择分辨率、刷新率、画质和带宽上限。例如 1080p / 30 FPS / 20 Mbps，或 1440p / 60 FPS / 50 Mbps。

### 场景 D：声音接收

用户在 Windows 控制 Mac 时，可以听到 Mac 端系统声音。控制端提供静音和音量控制，被控端提供是否允许采集声音的开关。

### 场景 E：剪贴板同步

用户可以在 Windows 复制文字，到 Mac 粘贴；也可以在 Mac 复制文字，到 Windows 粘贴。后续扩展到文件、压缩包、图片等内容。

### 场景 F：Mac 控制 Windows

用户在 Mac 上打开控制端，选择局域网内的 Windows，连接后在窗口中操作 Windows 桌面。

### 场景 G：一键反控

用户正在 Windows 控制 Mac 时，点击“反向控制”，Mac 端确认后切换为 Mac 控制 Windows。反过来也一样。

### 场景 H：离线局域网使用

两台设备断开互联网，但仍连接同一个路由器或交换机。用户可以通过 IP 地址直接连接。

## 4. 第一版范围

第一版只做必要能力：

- 中文主界面。
- 手动输入局域网 IP 和端口连接。
- Windows 控制 Mac。
- Mac 端采集屏幕。
- Windows 端显示远程画面。
- 鼠标移动、点击、滚轮、键盘输入。
- 连接密码或一次性配对码。
- 窗口化显示、缩放显示。
- 全屏显示开关。
- 基础分辨率选择。
- 基础刷新率选择。
- 基础带宽限制。
- 基础画质选项。
- 基础断线提示。

第一版可选增强：

- 控制端接收被控端声音。
- 文本剪贴板双向同步。

第一版暂不做：

- 公网穿透。
- 云账号登录。
- 多用户权限体系。
- 商业级硬件编码调优。
- 文件剪贴板。
- 大文件传输。
- 远程更新器。
- 复杂团队管理功能。

## 5. 第二版范围

- Mac 控制 Windows。
- 两端都能作为控制端和被控端。
- 一键反控。
- 局域网自动发现。
- 连接历史和设备备注。
- 文本剪贴板稳定同步。
- 文件、压缩包、图片等剪贴板内容传输。
- 多显示器选择。
- 更细的画质、分辨率、刷新率和带宽设置。
- 声音传输稳定化。

## 6. 第三版范围

- H.264/H.265 硬件编码。
- Opus 音频编码。
- 更低延迟输入链路。
- 自动重连。
- 安装包和开机自启。
- Windows 防火墙规则引导。
- macOS 权限引导页。
- 日志导出和故障诊断。

## 7. 用户体验原则

- 第一屏就是可操作界面，不做营销式首页。
- 所有按钮、提示、错误信息默认中文。
- 常用控制项集中在连接窗口工具栏：窗口化、全屏、分辨率、刷新率、带宽、画质、声音、剪贴板。
- 连接方式要透明：显示当前是局域网直连还是不可连接。
- 不隐藏安全风险：被控端必须明确显示“正在被控制”。
- 不默认开公网服务。
- 不默认允许无密码连接。

## 8. 安全原则

- 默认只监听局域网地址。
- 默认需要密码或一次性配对码。
- 配对信息只保存在本机。
- 控制连接使用加密通道。
- 被控端显示当前连接设备名称、IP 和连接时间。
- 被控端可单独关闭声音采集。
- 剪贴板同步默认可开关。
- 文件剪贴板默认需要接收方确认。
- 文件传输必须限制保存位置，不能允许远端任意写入系统目录。
- 一键反控必须由对端确认，不能静默切换。

## 9. 成功标准

第一阶段成功标准：

- Windows 可以在同一局域网内连接 Mac。
- 能看到 Mac 画面。
- 鼠标键盘控制可用。
- 控制窗口可以缩放、最小化、恢复。
- 全屏和窗口化切换可用。
- 分辨率、刷新率、带宽设置能生效，或明确显示当前版本暂不支持。
- 断网或关闭 Mac 端时，Windows 端能给出中文提示。

第二阶段成功标准：

- 控制端能听到被控端声音。
- 文字剪贴板能双向同步。
- Mac 可以反过来控制 Windows。
- 一键反控流程能跑通。
- 文件、压缩包等剪贴板内容能在两端安全传输。
- 两端都能通过同一套文档和协议继续开发。
