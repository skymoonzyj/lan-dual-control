- GitHub `main` 已同步到最新提交。
- Windows 控制端已支持中文界面、本地模拟、WebSocket 局域网连接、画质档位、分辨率、刷新率、码率、声音骨架、剪贴板骨架、一键反控状态机和桌面 exe 构建。

## 明天优先目标

1. 完成连接安全的第一轮实装。
2. 增加 Windows 侧一键自检和联调启动脚本。
3. 补强 Windows 被控端的基础可用性。
4. 更新任务清单并推送 GitHub，保证 Mac mini 到位后能直接接上。

## 任务 1：连接安全第一轮

目标：让“未授权设备不能连接”从任务清单里的空项变成可验证能力。

计划：

- 梳理当前 `auth_request`、`auth_result`、`LAN002` 的使用路径。
- 给假 Mac 服务和 Windows 被控端统一认证行为。
- 检查 Windows 控制端在密码错误、空密码、断线重连后的提示是否清楚。
- 记录连接密码暂不落盘的原则，避免本机保存明文密码。

验收：

- 密码错误时无法进入会话。
- 控制端显示明确中文错误。
- 本地模拟、假 Mac WebSocket、Windows 被控端骨架三条路径行为一致。

## 任务 2：一键自检和联调启动脚本

目标：明天以后不用手动记一串命令，减少反复折腾。

计划：

- 增加 Windows 侧脚本，用于检查 Node、npm、Tauri 构建产物和关键端口。
- 脚本能启动或提示启动这些组件：
  - Windows 控制端本地页面。
  - 假 Mac WebSocket 联调服务。
  - Windows 被控端骨架。
- 脚本输出中文结果：通过、失败、端口占用、依赖缺失。

验收：

- 一条命令能完成主要健康检查。
- 页面能加载到 1080P、2K、4K、Hz、Mbps 这些最新选项。
- 失败时能看到下一步该处理什么。

## 任务 3：Windows 被控端补强

目标：为后续 Mac 反控 Windows 提前铺路。

计划：

- 优先补文本剪贴板真实写入或更完整的接收状态。
- 评估 SendInput 输入注入的最小实现范围。
- 继续保持 Windows 被控端可以独立跑 `npm.cmd run check`。

验收：

- Windows 被控端日志能清楚显示收到的控制事件和剪贴板事件。
- 不破坏现有模拟视频帧、模拟音频帧和协议握手。

## 如果 Mac mini 明天到货

优先级临时切换为 Mac 真机对接：

1. 在 Mac 上安装 Git、Xcode Command Line Tools、Codex。
2. 克隆 GitHub 仓库。
3. 运行 `apps/mac-host` 的 Swift 骨架。
4. 验证屏幕录制、辅助功能、输入监控权限。
5. Windows 控制端连接真实 Mac 被控端，先看 hello/auth/session 是否跑通。
